# R-003: `/update-workflow` Command And Upstream Diffing

## Problem

Update behavior exists procedurally in initialization/update prompts, but no explicit first-class `/update-workflow` command path exists.

## Goal

Add `/update-workflow` for deterministic scaffold refreshes that can:
- prefer local scaffold sources
- fall back to GitHub upstream (`josiahH-cf/workflow`, `main`)
- compute/apply managed vs protected diffs
- clean temporary artifacts

## Expected Behavior

1. Discover local source first.
2. If missing, fetch upstream snapshot.
3. Normalize destination mapping.
4. Apply update rules from `meta-prompts/admin/update.md`.
5. Preserve protected files by default.
6. Return user to current phase or `/continue`.

## Candidate Files

- `meta-prompts/admin/update.md`
- `meta-prompts/admin/initialization.md`
- `template/AGENTS.md`
- `scripts/sync-prompts.sh`
- `prompts/` generated artifacts for new command

## Acceptance Criteria

- Command is discoverable in command maps.
- Source resolution order is documented.
- Managed/protected behavior matches existing update policy.
- Temporary download/checkout cleanup path is explicit.

## Validation

- Local-source update scenario succeeds.
- Upstream-fetch scenario succeeds.
- Protected files remain unchanged unless explicitly approved.
