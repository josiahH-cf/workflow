# Agentic End-to-End Test Runbook

> **Purpose:** Step-by-step execution guide for running the workflow from inception to completion on a new "Hello, world" Windows executable project. Mirrors the workflow's own phases. Includes tool adapters for Copilot, Claude Code, and Codex.
>
> **Prerequisites:** Read [`agentic_test_plan.md`](agentic_test_plan.md) first for architecture, scoring, and objective.

## Pre-Test Setup

These steps are performed **manually by the human** before handing off to the agent. They scaffold the project and inject the experiment observer.

### Step 0.1 — Install Cross-Compiler

```bash
sudo apt update && sudo apt install -y mingw-w64
# Verify:
x86_64-w64-mingw32-gcc --version
```

**If mingw-w64 is unavailable:** The agent will need to figure this out during Phase 4/6. That's a valid test of FAILURE_ROUTING. Proceed without installing and let the agent handle it.

### Step 0.2 — Create Target Directory

```bash
mkdir -p ~/hello-verify
cd ~/hello-verify
git init
git commit --allow-empty -m "Initial empty commit"
```

### Step 0.3 — Run install.sh

```bash
bash ~/workflow/scripts/install.sh ~/hello-verify/
```

**Expected output:** File counts for template, prompts, and meta-prompts. Post-install validation passes.

**Verify:**
```bash
bash ~/workflow/scripts/validate-scaffold.sh ~/hello-verify/
```

### Step 0.4 — Inject Experiment Observer

```bash
# Copy observer instruction file
cp ~/workflow/docs/verification/experiment_observer.md ~/hello-verify/workflow/EXPERIMENT_OBSERVER.md
```

**Patch AGENTS.md** — add observer row to Quick Reference table:

```bash
cd ~/hello-verify
# Add observer reference to AGENTS.md Quick Reference table
sed -i '/| Orchestrator state/a | Experiment observer rules              | `workflow/EXPERIMENT_OBSERVER.md` |' AGENTS.md
```

**Patch workflow/BOUNDARIES.md** — add ALWAYS rule:

```bash
sed -i '/- Reference the constitution for alignment/a - After every workflow action, append a structured entry to experiment-log.md per workflow/EXPERIMENT_OBSERVER.md' workflow/BOUNDARIES.md
```

**Patch CLAUDE.md** — add Session Bootstrap step 6:

```bash
sed -i '/5\. Read `workflow\/ORCHESTRATOR.md`/a 6. Read `workflow/EXPERIMENT_OBSERVER.md` — experiment logging rules' CLAUDE.md
```

### Step 0.5 — Create Initial Experiment Log

```bash
cat > ~/hello-verify/experiment-log.md << 'EOF'
# Experiment Log

> Auto-generated by the executing agent per `workflow/EXPERIMENT_OBSERVER.md`.
> Append-only. Do not edit previous entries.

**Test project:** hello-verify (Windows Hello World — C/mingw-w64)
**Started:** [TO BE FILLED BY AGENT]
**Workflow repo version:** [TO BE FILLED BY AGENT]
**Tool:** [TO BE FILLED BY AGENT]

---
EOF
```

### Step 0.6 — Commit and Open

```bash
cd ~/hello-verify
git add -A
git commit -m "Scaffold + experiment observer injection"
```

**Open `~/hello-verify/` as a new VS Code window.** The agent will work from this root.

### Step 0.7 — Verify Setup

Before proceeding, confirm:
- [ ] `~/hello-verify/AGENTS.md` exists and contains "Experiment observer" row
- [ ] `~/hello-verify/workflow/EXPERIMENT_OBSERVER.md` exists
- [ ] `~/hello-verify/workflow/BOUNDARIES.md` contains the logging ALWAYS rule
- [ ] `~/hello-verify/experiment-log.md` exists with header


   ```makefile
   CC = x86_64-w64-mingw32-gcc
   TARGET = hello.exe
   SRC = src/hello.c

   	$(CC) -o $(TARGET) $(SRC)

   test: $(TARGET)
   	file $(TARGET) | grep -q "PE32"
   	@echo "PE validation passed"

   clean:
   	rm -f $(TARGET)
   ```
4. **T-4:** Run `make` and verify tests pass

## Phase 6.5 — Bug Injection (Manual)

> **Purpose:** Test the debugging path by introducing a safe, deterministic build failure.
> **This step is performed by the human** after Phase 6 succeeds.

### Actions

**Introduce the bug:** Change the target triple in the Makefile from `x86_64-w64-mingw32-gcc` to `i686-w64-mingw32-gcc-posix-FAKE`:

```bash
cd ~/hello-verify
sed -i 's/x86_64-w64-mingw32-gcc/i686-w64-mingw32-gcc-posix-FAKE/' Makefile
git add Makefile
git commit -m "chore: update compiler target (introduces bug)"
```

**What this does:** The build will fail because the compiler binary doesn't exist. The error is:
```
make: i686-w64-mingw32-gcc-posix-FAKE: No such file or directory
```

**Why this bug:**
- It's a build failure (not runtime) — tests the `/bug` → `/bugfix` path
- The error message is clear — tests whether the agent reads it correctly
- The fix is obvious (revert the triple) — tests whether the agent fixes root cause vs symptom
- It mirrors the pattern of BUG-001 in workflow-example (a blocking build/runtime issue)

### Expected Agent Behavior
1. Next `/continue` or `/test post` detects build failure
2. Agent runs `/bug` to log it (severity: blocking)
3. Agent runs `/bugfix`:
   - **Reproduce:** Run `make`, observe failure
   - **Diagnose:** Identify wrong compiler binary name
   - **Fix:** Correct the target triple back to `x86_64-w64-mingw32-gcc`
   - **Verify:** `make` succeeds, tests pass
   - **Ship:** Commit fix with `Fix BUG-001:` prefix

### What to Log
- Bug detection, bug log entry, bugfix steps, root cause identification, fix applied, verification

## Session Chunking for `/continue`

The `/continue` orchestrator has a **max 10 transitions per session** safety valve. For this test:

- **Phases 1–5** (setup through planning): ~5 transitions — fits in one session
- **Phase 6** (code): 2–3 transitions (pre → implement → post) — fits in one session
- **Phase 6.5–7** (bug injection + fix): 2–3 transitions — may need new session
- **Phase 7b–8** (review + maintain): 2–3 transitions — fits in one session

**Between sessions:**
1. Agent writes state to `workflow/STATE.json`
2. `experiment-log.md` persists across sessions (it's a file, not chat context)
3. New session: agent reads AGENTS.md → STATE.json → constitution → task file → EXPERIMENT_OBSERVER.md → resumes

**No context re-injection needed.** Everything is in files.
