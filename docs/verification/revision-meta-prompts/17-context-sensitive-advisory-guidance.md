# Phase 17: Context-Sensitive Advisory Guidance

## Objective
Define always-on advisory language adaptation based on context signals. Enable the system to provide more specific, actionable suggestions as it gains more context about the project state, user behavior, and workflow history — while ensuring all guidance remains optional and non-blocking. Add language personalities that fit use cases as more context is available (e.g., "novice-friendly", "expert-level", "collaborative tone"). The goal is to make the system feel more helpful and personalized without ever forcing a particular style or recommendation on the user, and the ability to switch between different advisory styles as needed.

- `docs/verification/revision-spec/RS-017-context-sensitive-advisory-guidance.md`

## Deliverables
- Context signal and recommendation-goal guidance.
- Non-blocking advisory style expectations.

## Constraints

## Verification
- Guidance specificity increases with context quality while preserving user control.
