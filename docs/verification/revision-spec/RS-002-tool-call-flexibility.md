# RS-002 Tool Call Flexibility

## Goal
Establish a workflow policy where tool usage is not artificially restricted by internal docs or prompt scaffolding unless required by external policy.

## In Scope
- Identify and remove or neutralize internal hard tool constraints.
- Align prompt/meta-prompt language to default-open tool usage intent.
- Preserve safety and external policy compliance.

## Out of Scope
- Changing platform-level tool availability.
- Overriding non-project policy boundaries.

## Observable Outcomes
- No workflow artifact imposes avoidable tool limitations.
- Tool guidance emphasizes capability and context over static allowlists.

## Dependencies
- None.
